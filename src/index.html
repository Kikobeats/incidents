Track your incidents

incidents.now.sh/:id?[query]

source code

Usage

Create a new incident namespace just passing an unique identifier (e.g., /incidents). The first time you enter on the URL, it'll create a time stamp and it starts counting from that moment.

When you want to reset the counter, just pass `?reset` query parameter.

Query Parameters

The service support a log of customization that needs to be passed as query parameters, being all of them optional:

theme

default: light

values: light | dark

Select the theme style to use (e.g., /incidents?theme=dark).

text

default: since last incident

Customize incident copy (e.g., /incidents?text=since last ocurrence).

since

default: now

Provide relative timestamp (in a humanized way) for creating the time reference (e.g., /incidents?since=1d).

quote

Attach a message with your incident (e.g., /incidents?quote=all is working as expected).

json

When is present, the response format will be a JSON (e.g., /incidents?json).

reset

When is present, it reset the timestamp associated with the namespace (e.g., /incidents?reset).
